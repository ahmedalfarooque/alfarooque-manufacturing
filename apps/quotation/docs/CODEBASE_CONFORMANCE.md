# Codebase Conformance Addendum (v1.1)
**Supersedes the technical-stack details in `Quotation System Master Specification.md` §12–§19 and in `Claude_Code_Implementation_Prompt.md` wherever they conflict.** The functional design (modules, screens, costing formulas, workflows, roles, data model shape) is unchanged.

After inspecting the real monorepo at `alfarooque-manufacturing live`, the quotation app follows the **actual** conventions of its siblings `apps/cars` and `apps/projects`, not the assumptions made before the codebase was available:

| Topic | Spec originally assumed | Actual convention (now authoritative) |
|---|---|---|
| Location | separate repo `quotation-app` | **`apps/quotation`** inside the main-site monorepo; main static site untouched |
| Language | TypeScript strict | **JavaScript** (`.js`), `jsconfig.json` with `@/*` alias — same as cars/projects |
| Auth | Supabase Auth | **Custom platform auth**: shared `platform_users` / `platform_otp_codes` / `platform_sessions` tables, email+password (admin) or email OTP (user), JWT session cookie `af_quotation_session` (jose/jsonwebtoken, 12 h TTL), `APP='quotation'` scoping — cloned from `apps/projects/lib/auth.js` |
| DB access | RLS + anon key | **Service-role client server-side only** (`lib/db.js`), role checks in API routes via `lib/http.js` `requireSession` — same trust model as siblings |
| Schema management | Supabase CLI migrations | **Single idempotent SQL file** `supabase/quotation-schema.sql` applied via Supabase Dashboard → SQL Editor (same as `apps-schema*.sql`); all tables prefixed **`qt_`** for isolation in the shared Supabase project |
| i18n | next-intl, messages/*.json | **`lib/i18n.js` LanguageProvider** with inline EN/AR dictionaries, `localStorage('af-quotation-lang')`, RTL via `document.dir` — same API as projects |
| Theme | shadcn tokens | Existing glass design system (`app/globals.css` copied from projects), dark mode class + `localStorage('af-quotation-theme')`, brand Tailwind palette from `tailwind.config.js` |
| Icons | lucide | **GlassIcons sprite** (`public/glass-icons.svg` + `components/GlassIcons.js`) |
| PDF | @react-pdf/renderer | **jspdf + jspdf-autotable** (already used by cars); Excel via **exceljs**; charts via **recharts** |
| State/query libs | TanStack Query/Table, Zustand, RHF, Zod | Keep sibling minimalism: React state + small hooks (`useLiveData`, `useDebouncedValue`, `useSortableData`); add heavier libs only if a phase truly needs them (note in DECISIONS) |
| Deployment | new Supabase project | **Same Supabase project** as siblings; own Vercel project rooted at `apps/quotation` → **quotation.alfarooque.com** (see `apps/DEPLOYMENT.md`); dev port **3020** (cars 3010) |
| Email | Resend templates | `lib/email.js` Resend-over-HTTP pattern copied from projects (mock-OTP console fallback in dev) |
| RBAC | Postgres RLS policies | Coarse role on JWT (`admin`/`viewer`/`external`) enforced in middleware + `requireSession`; fine-grained permissions (costs.view etc.) implemented in API route guards reading `qt_settings`/role mapping — Phase 6 |

## Environment variables (`.env.local` in `apps/quotation`)
Same names the siblings use: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `JWT_SECRET`, `RESEND_API_KEY` (optional in dev — OTP prints to console), `EMAIL_FROM`.

## Phase status
- **Phase 0 (done in this scaffold):** app shell, login (admin password+OTP / user OTP), middleware, bilingual EN/AR + RTL, dark mode, dashboard with live KPI counts, notifications plumbing, placeholder pages for all modules, pure costing engine `lib/costing.js`, DB schema + seed (`supabase/quotation-schema.sql`).
- **Phase 1 (done):** master-data CRUD + UI for Customers, Suppliers, Materials & Hardware (kind tabs, category filter, price history), Labour (rate history), Machines, Expenses; shared `MasterList` component + `lib/crud.js` generic handlers with audit logging; Excel importers for the WW-03 Customer Details file and the Purchases Products Report (supplier upsert, material match/create, price-history append, latest-price update); full EN/AR labels.
- **Phase 2 (done):** Product Catalogue list + product detail with the six-tab Cost Model editor (Materials | Hardware | Labour | Machines | Expenses | Other): master-data pickers with snapshot lines, live cost summary (production → overhead → risk → profit → selling, inverse selling-price mode, rounding, low-margin warning), server-side authoritative rollup via the same `lib/costing` engine, Recost diff-and-apply, product duplicate, catalogue price history on "Set Standard Price". Requires `supabase/quotation-schema-v2.sql` (adds `qt_catalogue_products.cost_params`).
- **Phase 3 (done):** Quotations list (status tabs, search) + full editor: create per entity with race-safe numbering (`qt_next_quote_number`), customer typeahead, header terms/dates/PDF-language, products via **From Catalogue** (price + description + cost-model snapshot) or **Detailed Product** (inline six-tab CostModelEditor), per-line qty/unit/price/taxable, "use calculated selling price", live summary (Subtotal → Discount %/amount → Net → VAT → Grand Total + internal cost/blended margin), autosave every 2.5 s (draft-only, version-conflict guard, server-authoritative recalc in `/save`), submit with threshold check (auto-approve or pending_approval per `approval_thresholds` setting), approve/send-back, mark accepted/rejected with win-loss reasons, duplicate, and revisions (`-Rn`, source superseded, events logged).
- **Phase 4 (done):** `QuoteDocument` — bilingual (EN/AR + RTL) A4 quotation render matching the company's Excel layout (entity letterhead block, items table with multi-line descriptions, Subtotal → Discount → Net → VAT → Grand Total, terms, signature blocks). Internal **Print/PDF view** (`/quotations/[id]/print`, browser print-to-PDF for perfect Arabic shaping, language toggle). **Public read-only view** `/q/[public_token]` (no auth, customer-safe projection — no costs/internal notes; hidden for draft/cancelled). **Send Email** via Resend (`/api/quotations/[id]/send`): branded bilingual HTML with totals + secure link, approved → sent transition, event logged, mock mode without RESEND_API_KEY. **Expiry sweep** `/api/cron/expire` (Vercel Cron daily 03:00 via vercel.json + CRON_SECRET, or manual admin call): approved/sent past validity → expired + notification to salesperson. Editor gained Print/PDF, Send Email, Copy Link actions.
    - Design note: PDF is produced by the browser's print engine rather than jspdf — jspdf cannot shape connected Arabic script reliably; print-to-PDF renders both languages perfectly and keeps one document component for print + public views. Server-generated PDF attachments can be added later (headless Chromium) without changing the document component.
- **Phase 5+6 (done):**
  - **Schema v3** (`supabase/quotation-schema-v3.sql`): product sub_category/sku/barcode/dimensions/images/notes, sequence-backed never-reused product codes (`qt_next_product_code`), quotation-product dimensions, supplier country/currency/bank/IBAN/contacts, `qt_user_roles`. **Demo-data cleanup**: `supabase/quotation-cleanup-demo-data.sql` (review before running; keeps structure & seeds, resets numbering).
  - **Auto-translation** (`lib/translate.js`): offline EN↔AR industry dictionary with word-order reversal; wired server-side into all create/update handlers (name/company/description pairs) — enter one language, the other is generated, both stored, manually editable.
  - **Products**: ERP catalogue — Category/Sub-Category, SKU/barcode, optional dimensions (L/W/H/thickness/depth/diameter/weight/area/volume), multi-image drag-drop gallery with primary (public `product-images` bucket), archive/restore, thumbnails + bilingual names + updated column in list.
  - **Dynamic size pricing**: `scaleFactor`/`scaleCostLines` in the costing engine (volume → area → length basis); editing dimensions on a quotation line rescales material/hardware/labour/machine quantities from the base snapshot and re-derives the price instantly. **Save as New Product** button + submit-time prompt clones changed specs to a new catalogue product (original untouched).
  - **Import/Export/Templates**: products, suppliers, materials — xlsx/csv import with row-level validation + error report, full export, downloadable templates with example rows (`/api/export/{table}`); materials **bulk price update** (%/fixed, preview → apply, history recorded).
  - **Reports** (10): quotation register, sales by customer, profit & margin, VAT summary, products price list, material price moves, top materials, labour, machines, expenses — filterable, Excel/CSV export, PDF via print.
  - **Dashboard**: 12-month quoted-vs-accepted chart (recharts) + recent quotations.
  - **Users & roles**: qt roles (admin/manager/sales/estimator/accountant/production/readonly) assigned per user, permission matrix UI; enforcement via `requireWrite`/`can` in every mutating route; approvals restricted to admin/manager.
  - **Settings**: Entities (names/CR/VAT/address/prefix/next number), Defaults & VAT, Profit, Approval thresholds, Numbering, auto-translation toggle, System (env-var reference for Resend/backup/storage).
  - **Audit viewer**: filterable log with before/after JSON diff.
- **Language system redesign (single-language data model):** every bilingual field pair (name_en/name_ar, description_en/ar, company_name pair, terms body pair, cost-line names) replaced by ONE canonical column via `supabase/quotation-schema-v4-single-language.sql` (full backup of all pairs kept in `qt_lang_backup` before dropping; original-language value wins on merge). Display-time auto-translation: `tr()` in the LanguageProvider renders every stored value in the current UI language (in-memory + sessionStorage cache, batched `/api/translate` backed by the shared `qt_translations` server cache and the dictionary engine — swap in a real MT API in one place). Bilingual search: queries are translated and OR-matched, so "Premium" and "باب" both find the same record. Documents (`QuoteDocument` print/public/PDF), reports, exports and templates localize headers AND stored values to the requested language (`?lang=`), Arabic fully RTL. English is the default; preference persists. EXCEPTION: `qt_entities` keeps registered legal names/addresses in both languages — letterheads must print exactly as registered, never machine-translated. Legacy import headers (Name EN/AR) still accepted by importers.
- **Known design choices:** translation is deterministic dictionary-based (no external API — extend `lib/translate.js` PAIRS or plug an API later); PDFs use the browser print engine (correct Arabic shaping); legacy `.xls` must be re-saved as `.xlsx` (csv/xlsx supported natively).
